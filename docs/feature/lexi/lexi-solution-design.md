# Lexi Solution Design
**Version:** 1.0
**Created:** 2026-02-13
**Status:** Tier 1 & 2 Complete, Integration In Progress
**Author:** Tutorwise Product Team

---

## 1. What Is Lexi

Lexi is a conversational AI assistant that provides contextual help to all Tutorwise users. It acts as a unified interface to the platform's capabilities, routing requests through role-specific personas to deliver personalized assistance.

Lexi is **not** a general-purpose chatbot. It is a purpose-built assistant that:

- Understands user context (role, permissions, current state)
- Routes conversations through 5 distinct personas (Student, Tutor, Client, Agent, Organisation)
- Delegates complex operations to CAS (Conversational Agent System) agents
- Maintains conversation history for continuity and analytics
- Respects rate limits and session boundaries

---

## 2. Core Principle

```
One interface, many personas.
Context-aware routing.
Permission-checked operations.
Never expose admin capabilities to users.
```

Lexi appears as a single assistant but internally routes to different personas based on the user's role. Each persona has specific capabilities and tone appropriate for its audience.

## 5. Persona Capabilities

### Student Persona
- Learning assistance and homework help
- Progress tracking and achievements
- Lesson scheduling and reminders
- Resource recommendations

### Tutor Persona
- Schedule and availability management
- Earnings and payout tracking
- Student progress monitoring
- Resource creation tools

### Client Persona (Parent/Guardian)
- Tutor search and discovery
- Booking management
- Child progress reports
- Payment and billing help

### Agent Persona
- Support ticket coordination
- Booking issue resolution
- Tutor assistance
- User lookup (own assigned users only)

### Organisation Persona
- Team analytics and performance
- Tutor management
- Compliance monitoring
- Revenue tracking

## 11. CAS User API Integration

Lexi delegates complex operations to the CAS User API modules:

### Learning Module
- `getRecommendedResources()` - Personalized learning resources
- `getHomework()` - Homework assignments
- `searchResources()` - Resource search
- `generateStudyPlan()` - AI-generated study plans
- `explainTopic()` - Topic explanations

### Booking Module
- `searchTutors()` - Find tutors
- `getTutorAvailability()` - Check availability
- `createBooking()` - Create bookings
- `getBookings()` - View bookings
- `rescheduleBooking()` - Reschedule
- `cancelBooking()` - Cancel bookings

### Progress Module
- `getStudentProgress()` - Student metrics
- `getTutorAnalytics()` - Tutor performance
- `getOrgAnalytics()` - Organisation metrics
- `submitFeedback()` - Lesson feedback
- `getActivityFeed()` - Recent activity

### Key Security Rules

1. **Context Mode**: Lexi always operates in `'user'` mode, never `'admin'`
2. **Row Level Security**: PostgreSQL RLS policies restrict data access
3. **Rate Limiting**: Prevents abuse at the Lexi layer
4. **Session Isolation**: Users can only access their own sessions
5. **Audit Logging**: All operations are logged with context
